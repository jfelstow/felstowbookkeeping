How it works

Your Monday, per client: about 15 minutes.

1. Bank balance in

The client texts you their Monday bank balance. You drop it in a one-line text file (or pre-stage it and let the 8 AM cron handle everything).

2. The system publishes

The pipeline rolls the 13-week window, recomputes every total in Python, encrypts the dashboard, and pushes it live — in about 60 seconds.

3. You advise

You read the dashboard before they do and send three sentences: the verdict, the number that matters, the thing you’re watching. That email is the retention engine.

FAQ
Do I need to know how to code?
No. The pipeline scripts work as-is after filling in a clearly marked config block. The two files that are custom per client (the spreadsheet math and the dashboard branding) come with copy-paste AI prompts that do the translation for you — plus a verification step so you know the math is right.
What do I need?
A Mac, Python 3, Node.js, and a free GitHub account. Total infrastructure cost: $0/month.
Is this a course?
No. It’s a working system — the same scripts and playbook running for real clients today — with documentation written so you can run it too.
Why is the hosting free if the data is sensitive?
The published file is AES-256 encrypted ciphertext; the password your client enters decrypts it in their browser. Nothing readable ever sits on the host.
Does it work with QuickBooks?
The system runs from a 13-week cash flow spreadsheet (included structure), which you update from any source — QBO, bank feeds, or the client’s Monday text. The bank balance, not the book balance, drives it: that’s rule #7 in the Blueprint.
